The problem

Respo Financial's customer support team was spending most of its time on repetitive, templated tickets — EMI queries, account status checks, repayment schedules, and escalation acknowledgements. Response times averaged 48 hours, leading to customer churn and agent burnout.

The approach

Built a GraphRAG pipeline using a Neo4j knowledge graph that models relationships between customers, products, policies, and past interactions. Incoming tickets are embedded and matched against the graph, the relevant subgraph is retrieved, and a local LLM (served via Ollama) drafts the response. A LangGraph orchestrator handles routing, human-escalation detection, and workflow automation.

Impact

77%of tickets resolved fully autonomously
48h→1mfirst-response time

The remaining 23% are routed to agents with a pre-populated context summary, cutting handling time in half for those too.

Architecture highlights

GraphRAG outperforms flat vector search here because relationships matter — an EMI query needs the customer's account graph, not just keyword similarity. The system runs entirely on-prem with Ollama (Mistral 7B), so no PII leaves the perimeter. LangGraph provides the retry/escalation loop with full observability.

Technology stack

GraphRAG Neo4j LangGraph Ollama (Mistral 7B) Python FastAPI Pinecone Docker

Other AI work at Respo Financial

Alongside the GraphRAG system, I built AI voice-calling agents using Sarvam STT/TTS for debt collection and EMI reminders — completing 3,000+ automated calls per day, equivalent to 30+ human agents. Operational dashboards track call outcomes, escalation rates, and bot performance.
